# Documentation

Welcome to the CleanStack documentation, a modern Next.js starter implementing Clean Architecture and Domain-Driven Design.

## About CleanStack

CleanStack is a Next.js template designed for AI-assisted development. It combines Clean Architecture and Domain-Driven Design with comprehensive documentation that enables AI assistants (Claude Code, Cursor) to generate production-ready code instantly.

## Core Concepts

CleanStack is built on 3 pillars:

### 1. Clean Architecture

Separation into 3 layers with strict dependency rules:

```
Adapters â†’ Application â†’ Domain
```

Dependencies always point **inward**. The Domain has zero external dependencies.

[Learn more about architecture â†’](/docs/architecture)

### 2. Domain-Driven Design

Domain modeling with tactical patterns:

- **Value Objects** - Immutable business concepts (Email, Money)
- **Entities** - Objects with identity (User, Order)
- **Aggregates** - Consistency boundaries (UserAggregate)
- **Domain Events** - Decoupled asynchronous communication

[Discover DDD patterns â†’](/docs/core-concepts)

### 3. Functional Programming

Patterns for safety and maintainability:

- **`Result<T, E>`** - Explicit error handling without exceptions
- **`Option<T>`** - Absence of value without `null`
- **Immutability** - Domain objects frozen with `Object.freeze()`
- **Composition** - Combining results

[Understand Result and Option â†’](/docs/core-concepts/result)
