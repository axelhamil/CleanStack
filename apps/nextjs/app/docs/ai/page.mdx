# AI Development

CleanStack is optimized for AI-assisted development.

## Why AI-Friendly?

### Repeatable Patterns
AI recognizes DDD patterns instantly:

```typescript
// AI knows this is a Value Object
export class Email extends ValueObject<EmailProps> {
  protected validate(props): Result<EmailProps> {
    // Generates correct validation
  }
}

// AI knows this is a Use Case
export class CreateUserUseCase implements UseCase<Input, Output> {
  async execute(input): Promise<Result<Output>> {
    // Generates correct orchestration
  }
}
```

### Explicit Architecture
AI knows **exactly** where to place files:

```
domain/       ← Value Objects, Entities (zero deps)
application/  ← Use Cases, Ports
adapters/     ← Controllers, Repositories
```

### Strict Types
TypeScript + Result + Option = AI can't cheat:

```typescript
// ❌ AI can't generate this (TypeScript error)
const user = await userRepo.findById(id)
console.log(user.email)  // Error: user is Result<Option<User>>

// ✅ AI must generate this
const result = await userRepo.findById(id)
if (result.isFailure) return Result.fail(result.error)
if (result.value.isNone()) return Result.fail('Not found')
const user = result.value.unwrap()
```

## Example Prompts

**Create feature:**
```
Create a Product feature:
- Value Object: ProductName (3-100 chars)
- Entity: Product (name, price, isActive)
- Use Case: CreateProductUseCase
- Repository: IProductRepository + DrizzleProductRepository
- Route: POST /api/products
- Register in DI container
```

**Add validation:**
```
Add canPurchase() method to User that checks:
1. User is active
2. User has verified email
Return Result<boolean> with explicit error message.
```

**Refactoring:**
```
Rename Email to EmailAddress everywhere:
- domain/user/
- application/use-cases/
- adapters/out/
- Tests
```

## Best Practices

### ✅ Do
```
"In the domain layer, create..."
"Create a Use Case that returns Result<User>"
"Generate tests for this Value Object"
```

### ❌ Don't
```
"Add a user"              // Where? How?
"In domain, call the DB"  // Violation!
"Return the user"         // Result? Option? null?
```
