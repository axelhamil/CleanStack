# Cursor

Great for quick edits and autocomplete.

## Setup

1. Download from [cursor.sh](https://cursor.sh)
2. Open project: `cursor .`
3. Settings → Enable "Codebase indexing"
4. Model: Claude 3.5 Sonnet

## Shortcuts

### Cmd+K - Quick Edit
Select code → Cmd+K:
```
"Add validation for phoneNumber"
"Convert to Result pattern"
"Add unit tests"
```

### Cmd+L - Chat
```
"Explain the Dependency Rule"
"Where to create a new Value Object?"
"Create CreateProductUseCase like the project pattern"
```

### Tab - Autocomplete
Type class name → Cursor suggests structure based on patterns.

## Useful Prompts

**Architecture:**
```
@CLAUDE.md Explain the dependency rules
Where to create a Product repository?
```

**Code generation:**
```
Create DrizzleProductRepository implementing IProductRepository.
Use DrizzleUserRepository pattern.
```

**Multi-file edit:**
```
Rename Email to EmailAddress in:
- domain/user/
- application/use-cases/
- adapters/out/
```

## .cursorrules

Create at project root:

```
# CleanStack Rules

## Dependencies
- domain/ cannot import from application/ or adapters/
- application/ cannot import from adapters/

## Patterns
- Value Objects: extend ValueObject, implement validate()
- Entities: extend Entity, factory create(), use get()
- Use Cases: implement UseCase, return Result<T>

## Error handling
- NEVER throw in domain/ or application/
- ALWAYS use Result<T> and Option<T>
```

## Claude Code vs Cursor

**Cursor:** Excellent autocomplete, good multi-file editing, medium architecture understanding, no commits

**Claude Code:** Basic autocomplete, excellent multi-file editing, excellent architecture understanding, creates commits

**Recommendation:** Use both - Cursor for daily dev, Claude Code for complex features.
