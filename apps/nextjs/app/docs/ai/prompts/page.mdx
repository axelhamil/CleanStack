# AI Prompts

Ready-to-use prompts for CleanStack.

## Value Objects

```
Create Value Object [Name] in domain/[context]/:

Props: { value: [type] }

Validation:
- [rule 1]
- [rule 2]

Return Result<[Name]>.
Add tests in __TESTS__/.
```

**Example:**
```
Create Value Object ProductName in domain/product/:

Props: { value: string }

Validation:
- Not empty after trim
- 3-100 characters
- No special chars except space and dash

Add tests.
```

## Entities

```
Create Entity [Name] in domain/[context]/:

Props:
- [prop1]: [ValueObject1]
- [prop2]: [type]

Factory create() with validation: [rules]

Methods:
- [method1](): Result<[Name]>
- [method2](): boolean

Use get() for props, clone() for updates.
Add tests.
```

## Use Cases

```
Create [Name]UseCase in application/use-cases/:

Input: { [fields] }

Logic:
1. Validate and create VOs
2. Create Entity
3. Persist via I[Entity]Repository
4. Return Result<[Entity]>

Accept trx?: Transaction.
Add tests with mock repository.
```

## Repositories

```
Create I[Entity]Repository in application/ports/:
- Extends BaseRepository<[Entity]>
- Custom methods: [list]

Create Drizzle[Entity]Repository in adapters/out/:
- Implements interface
- Uses [Entity]Mapper
- All methods accept trx?: Transaction
```

## Routes

```
Create POST /api/[resources] in adapters/in/api/:

Zod schema: { [fields] }

Execute [Name]UseCase via DI.
Transform Result → HTTP:
- Success → 201
- Validation fail → 400
- Not found → 404
```

## Complete Feature

```

- Tests

- use-cases/Create[Entity]UseCase.ts
- Tests with mocks

**Adapters** (adapters/)
- Schema (packages/drizzle/src/schema/)
- mappers/[Entity]Mapper.ts
- persistence/Drizzle[Entity]Repository.ts
- api/[resources]/route.ts

**Infrastructure**
- DI module
- Migration
```
